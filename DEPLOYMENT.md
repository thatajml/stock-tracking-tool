# GitHub Upload & Deployment Guide

This guide explains how to upload the Stock Tracking Tool to GitHub and deploy both the Flask backend and the Vite + React frontend.

---

## 1. Uploading to GitHub

We have already initialized Git locally, set up a `.gitignore` to prevent tracking of unnecessary or sensitive files (like `venv`, `node_modules`, `chromedriver`, logs, etc.), and made an initial commit. 

To upload this to your GitHub account:

### Step 1: Create a Repository on GitHub
1. Go to [GitHub](https://github.com) and log in.
2. Click the **New** button (or go to `https://github.com/new`).
3. Set the repository name to **`StockTrackingTool`** (or any name you prefer).
4. **Crucial:** Leave "Add a README file", "Add .gitignore", and "Choose a license" **UNCHECKED**. (We already have these configured locally, and initializing them on GitHub will cause merge conflicts).
5. Click **Create repository**.

### Step 2: Link Local Repository and Push
Open your terminal (inside the root directory `/Users/Allu/SEM/StockTrackingTool`) and run the following commands (replace `<your-username>` with your actual GitHub username):

```bash
# Add the remote GitHub repository
git remote add origin https://github.com/<your-username>/StockTrackingTool.git

# Set the default branch name to main
git branch -M main

# Push the code to GitHub
git push -u origin main
```

## 2. Deploying the Backend (Flask)

We will use **Render** (free tier) to host the Python Flask backend.

### Step 1: Sign up for Render
1. Go to [Render](https://render.com) and sign up/log in (you can sign in using your GitHub account).

### Step 2: Create a New Web Service
1. Click **New +** in the top right, and select **Web Service**.
2. Connect your GitHub repository (`StockTrackingTool`).
3. Set the following configuration details:
   - **Name:** `stock-tracking-backend` (or similar)
   - **Language:** `Python`
   - **Root Directory:** `backend`
   - **Branch:** `main`
   - **Build Command:** `pip install -r requirements.txt`
   - **Start Command:** `gunicorn app:app`
   - **Instance Type:** `Free`
4. Click **Create Web Service**.

> [!NOTE]
> Render's free tier services will spin down if they are inactive for 15+ minutes. If the backend is cold, the first request from your frontend might take 30-50 seconds to respond as it spins back up.

## 3. Deploying the Frontend (Vite + React)

We will use **Vercel** or **Netlify** to deploy the frontend. Both are fast, free, and support automatic deployments on push.

### Option A: Deploying on Vercel (Recommended)
1. Go to [Vercel](https://vercel.com) and log in using your GitHub account.
2. Click **Add New...** -> **Project**.
3. Import the `StockTrackingTool` repository.
4. Set the following configuration details:
   - **Framework Preset:** `Vite` (Vercel detects this automatically)
   - **Root Directory:** Edit this and select the **`frontend`** directory.
   - **Build and Output Settings:** (Keep defaults: Build command `npm run build`, Output directory `dist`)
   - **Environment Variables:** Expand this section and add:
     - **Name:** `VITE_API_URL`
     - **Value:** `https://<your-backend-name>.onrender.com/api` (use the URL generated by Render for your backend Web Service, making sure it ends in `/api`).
5. Click **Deploy**.

### Option B: Deploying on Netlify
1. Go to [Netlify](https://netlify.com) and log in using your GitHub account.
2. Click **Add new site** -> **Import an existing project**.
3. Select **GitHub** and connect your `StockTrackingTool` repository.
4. Set the following configuration details:
   - **Base directory:** `frontend`
   - **Build command:** `npm run build`
   - **Publish directory:** `frontend/dist`
   - **Environment Variables:** Add `VITE_API_URL` with your Render backend API URL.
5. Click **Deploy site**.

## 4. Local Development

You can continue working on this project locally. The configuration remains backward compatible:

1. **Start the backend** (port `5000` by default):
   ```bash
   cd backend
   source venv/bin/activate
   python app.py
   ```
2. **Start the frontend**:
   ```bash
   cd frontend
   npm run dev
   ```
   The frontend will automatically fall back to `http://127.0.0.1:5000/api` if no `VITE_API_URL` environment variable is defined.
